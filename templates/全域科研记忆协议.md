# 全域科研记忆协议 v1.0

## 适用范围

本协议适用于所有 AI agent，包括 Codex、Pi agent、Claude、Cursor、Gemini、MCP agent 或其他自动化工具。

无论当前工作目录是否位于 Obsidian Vault，只要任务属于科研、论文、数据分析、图表、投稿、文献整理或项目管理，都应遵守本协议。

## 核心原则

Obsidian Vault 是正式科研记忆库和 source of truth。

Agent 内部记忆、聊天记录、临时缓存、Supermemory、agentmemory 或其他 MCP memory 只能辅助召回，不能替代 Project Hub、Registry 和长期数据库。

## 最小读取规则

识别项目后，默认只读取：

- `00_Project Hub.md`
- `00_Project Memory.md`，如存在
- `00_Existing Project Intake.md`，如存在
- 本任务直接相关的 Registry

不要默认全文读取旧材料、PDF、脚本、原始数据、图表或历史草稿。

## 深读规则

旧材料是冷存储。只有当前任务确实需要证据核验、原文确认、数据追溯、脚本检查、图表重画或稿件修改时，才深入读取对应源文件。

深入读取后，必须更新对应 Registry 或材料摘要。

## 图表版本规则

图表文件不默认进入长期记忆库，长期记忆库默认只保存图表元数据。

Agent 不得自动删除图表文件，除非用户明确批准清理。

图表更新时，应尽量新建版本而不是覆盖旧版本，并同步更新 `Figure Registry.md` 和 `Version Index.md`。

图表状态使用以下值：

| Status | Meaning |
|---|---|
| `current` | 当前推荐版本 |
| `candidate` | 候选版本，仍在比较 |
| `superseded` | 已被替代，但短期可回滚 |
| `rejected` | 明确不采用，清理仍需用户确认 |
| `archived` | 关键历史版、投稿版或不可复现版本，长期保留 |
| `missing` | Registry 中记录存在，但本地未找到 |
| `deleted` | 文件已被用户或历史流程删除，仅保留记录 |

## 任务结束写回规则

每次科研任务结束时，默认写回工作记忆。

允许自动写回：

- Task Summary
- 读取了哪些材料
- 创建或修改了哪些文件
- 当前推荐版本
- 待核验项
- 证据缺口
- 阻塞点
- 下一步行动
- AI 协作记录
- 版本记录
- Registry 中的文件状态更新

未经用户明确确认，不得自动写成“已验证结论”：

- 研究主结论
- 统计解释
- 医学结论
- 文献证据等级
- 可投稿判断
- 正式采用或废弃某个结果

## 长上下文防遗忘规则

如果会话很长、跨越多轮任务，或即将写入记忆文件，agent 应先重新读取：

- Bootstrap
- 目标项目的 `00_Project Hub.md`
- 目标项目的 `00_Project Memory.md`
- 本次要更新的 Registry

不得只依赖聊天上下文做最终写入。

## 科研诚信底线

- 不编造数据、文献、统计结果、医学结论或机制结论。
- 不把相关性写成因果性。
- 不把假设写成事实。
- 不使用 simulated、synthetic、assumed 或 AI-generated 数据作为正式研究证据。
- 不确定内容标注为 `需要核验`、`待补证据`、`blocked` 或 `暂不采用`。
